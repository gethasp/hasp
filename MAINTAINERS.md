# Maintainers

This file explains who owns the public repo surface and what contributors
should expect.

## Current maintainer

- `ehmo`
  - owns the public broker release path
  - reviews repo-shape and product-surface changes
  - handles security intake through the private GitHub flow

## What maintainers are expected to do

Maintainers should:

- keep the public docs current
- keep the documented install and verification path working
- review public issue intake
- keep release notes, support guidance, and security policy readable

## Review expectations

Small bug fixes and docs fixes should move quickly.

Broader product changes should arrive with:

- a linked issue
- tests or a clear verification note
- docs updates when the user-facing surface changed
